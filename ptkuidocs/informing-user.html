Informing User

Many a times it is required to inform the user about the action he has performed.

Types of messages

  • In-form messages
  • In-page messages
  • Popup messages

In-form messages

These messages are displayed in response to user input in form fields.

The messages that apply for form fields should be displayed as described below. All form field messages should appear either right next to field or right below the field.

The messages that apply for only form should be displayed between last field of the form and actions buttons of the form.

Message type Description
Suggestion This message indicates a suggestion.
Affirmative This message indicates that the fields value is acceptable.
Warning This message indicates a warning.
Error This message indicates an error.

In-page messages

These messages are shown to the user in response to the action he/she has taken.

These messages should be displayed after the navigation bar on top of the page. The page should be scrolled to the message to bring the message in user view.

The examples below are only suggestions on how a message can look. These examples are inspired from Bootstrap library.

On clicking the × in the top right corner of the message, message must become hidden from the user.

Message type Description
Info
This message is just for your information.
Success
You were successful, whatever it is you were doing.
Warning
This is a warning in repsone to your action.
Danger
Something bad happened.

Popup messages

Popup messages are displayed to the user by showing some content in the box.

A popup message may have a header, a set of action buttons and a message text.
